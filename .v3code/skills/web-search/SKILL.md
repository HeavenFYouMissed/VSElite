# Web Search Skill

When using `web_search`, follow these rules strictly:

## Query Formation
- Use SHORT keyword queries (2-5 words maximum)
- NEVER use natural language sentences as queries
- Split complex topics into multiple small searches
- If a query returns nothing, make it shorter/simpler

## Examples

WRONG: "how do I implement a websocket server with authentication in Node.js using the ws library"
RIGHT: "node ws library authentication"

WRONG: "what is the latest version of React and how do I upgrade"
RIGHT: "react latest version 2025"

WRONG: "best practices for error handling in TypeScript async functions"
RIGHT: "typescript async error handling"

## Strategy
1. Start with the most specific 2-3 word query
2. If no results, broaden slightly
3. If still nothing, try alternative terminology
4. Use results from first search to inform follow-up queries

## Source Evaluation
- Prefer official documentation over blog posts
- Prefer recent content (check dates) over old articles
- Cross-reference multiple sources for accuracy
